Референси — візуальна мова інвойсингу

Реальна візуальна мова бенчмарку, вже названого в дослідженні (Harvest, Paymo, Clockify) — знайдена через MCP-сервер Refero, без нового пошуку конкурентів. Один стиль за основу, 1–2 конкретні прийоми з інших — не «загальне враження».

Деталі — у concept/references.md · Правило синтезу — CLAUDE.md §9 · Бенчмарк — research.html §1

01Пошук у Refero

Два описові запити (не назви конкурентів): стилі — «мінімалістичний, професійний SaaS-інструмент для фрілансерів, довірлива» (desktop-перший — 3/3 інтерв'ю трекають з комп'ютера, jtbd.md); екрани — «статус-бейдж оплати, картка інвойсу з довірою».

Стилі — топ-3, повні дані отримано

Екрани — топ-3

02Синтез — обрано за основу

Жоден референс не копіюється цілком (CLAUDE.md §9). Один стиль — фундамент, 1–2 прийоми з інших — точково.

Основа: Acctual

Ахроматична паперово-біла база (#ffffff / #000000 / #8d8d8d) з одним яскравим акцентом Sky Teal #0098f2, застосованим виключно до інтерактиву — майже дослівно правило CLAUDE.md §9. Готовий компонент Invoice Card: радіус 20px, тінь rgba(0,0,0,.06) 0 2.5px 2.5px — база для наших карток інвойсів/звітів (узгоджено з «rounded elements», коміт 97fdce4).

Свідомо НЕ переносимо: декоративні Hot Pink/Vivid Violet і рукописний Caveat — прикраси лендингу, другий/третій колір порушив би правило «один акцент».

З Raise (Open Collective)

  • Тонка 1px рамка #e2e8f0 замість тіні — для розділення карток у щільних списках (звіти, таблиці записів), де тінь на кожному рядку була б «важкою».
  • Приглушений сіро-синій чіп #90a1b9 — зразок нейтрального тону для другорядних/неактивних станів («архівовано»).

З Perplexity AI

  • Активний пункт sidebar — тонований нейтральний фон #FAF8F5, не акцентний колір, радіус 8px.
  • Підтверджує вже прийняте «лівий app-nav» (коміт 97fdce4): активний стан через surface-tint, акцент лишається тільки для дій.

03Статус-бейдж не сплачено · відправлено · сплачено

Два контексти — два різні місця, обидва підтверджені реальними екранами (CLAUDE.md §8).

Документ інвойсу (детальний перегляд)

Runey кладе статус-чіп («Paid») у верхньому хедері, поруч із чіпами номера інвойсу та проєкту — статус видно ще до відкриття тіла документа.

→ Прийом: бейдж у header-рядку поряд з номером/клієнтом, не в таблиці рядків.

Список інвойсів (таблиця/картки)

GlossGenius кладе статус як заокруглений pill-бейдж в окрему колонку «Status», між назвою і сумою.

→ Прийом: окрема колонка/зона під статус, відокремлена від суми.

Радіус: шкала Acctual — бейджі повністю заокруглені (pill), як і кнопки — узгоджено з «rounded elements». Колір: GlossGenius одночасно змішує brand-blue (банер), status-green (бейдж) і decorative-pink (аватар) — 3 кольори поза нейтральною базою. Це НЕ переноситься: статусні кольори мають бути приглушені й нечисленні (2–3 тихі відтінки), окремі від єдиного бренд-акценту.

04Скільки кольору на екрані

Acctual і Runey — обидва ~95% ахроматика + рівно один семантичний акцент. Це цільова пропорція для TrackOn.

Canvas white Ink / graphite Muted gray Один акцент (інтерактив) project.color — лише мітка приналежності
Явний анти-приклад: GlossGenius і «тепла» мова конкурентів із research.html §1.4 (кожен проєкт/клієнт — свій колір, аватарки в рожевому, банери в синьому) — саме те, що CLAUDE.md §9 просить не переносити дослівно. project.color — єдиний дозволений «додатковий» колір поза акцентом, лише як мітка приналежності (плитка/бордюр), не фон великих поверхонь.

05Джерело × прийом × тривога персони

Для кожного джерела — конкретний прийом і яку тривогу персони він знімає (jtbd.md).

ДжерелоПрийом (конкретний)Тривога персони
Acctual Ахроматична база + 1 акцент лише для інтерактиву; Invoice Card 20px радіус, тінь 0.06 opacity E2 — «не загубитися у складності»: відсутність зайвого кольору й ваги = не «CRM»-відчуття, що змусило кинути Harvest/Clockify
Runey Статус-чіп у хедері документа поруч з номером/проєктом; серифна типографіка + рукописний підпис S1 — «виглядати професійним перед клієнтом» і E1 — «почуватися чесним у цифрах»: статус видно миттєво, документ не соромно надіслати
GlossGenius Pill-бейдж статусу в окремій колонці списку, відокремлений від суми E2 — швидке сканування списку без перевантаження (їхню 3-колірну систему НЕ переносимо — порушує CLAUDE.md §9)
Raise / Open Collective Тонка 1px рамка замість тіні; приглушений сіро-синій чіп для неактивних станів E1 / E3 — довіра через «бухгалтерську» точність без декоративності; прийом для щільних звітних таблиць
Perplexity AI Активний пункт sidebar — тонований нейтральний фон, не акцентний колір E2 — узгоджує вже прийняте «лівий app-nav», тримає акцент виключно для дій

+Що це означає для наступного hi-fi кроку

  • Базова палітра: білий/графіт/сірий + один ще не обраний бренд-акцент (відкрите питання CLAUDE.md §12).
  • Радіус карток/інвойсів: ~16–20px, тінь легка (~0.06 opacity) — уникати важкої елевації рівня Raise.
  • Бейдж статусу: повністю заокруглений (pill); два різні місця залежно від екрана.
  • Статусні кольори — 2–3 тихі відтінки, окремі від бренд-акценту.
  • project.color — єдиний додатковий колір поза акцентом, лише мітка приналежності.
