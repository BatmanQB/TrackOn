# Правила вайрфреймів — TrackOn

Основа для **всіх** екранів. Спираються на [`../sitemap.md`](../sitemap.md) (структура, ієрархія, стани)
і [`_screens.md`](_screens.md) (екрани головного flow та їхні реальні стани).
Домен — **трекінг часу для фрілансера** (проєкти, клієнти, записи часу, ставки, інвойси), не lorem.

---

## 1. Деталізація — low-fi, тільки каркас
- Тільки **структура, ієрархія і зони**. Жодного візуального дизайну.
- **Лише відтінки сірого:** фон білий/світло-сірий, зони — сірі блоки, рамки `#ccc`, текст темно-сірий.
  Колір як засіб **не використовуємо** (навіть акцент бренду).
- **Без:** шрифтів (системний дефолт, жодного `font-family`), бренду, логотипів, **картинок**.
- Медіа-зони (напр. колір-плитка проєкту, аватар клієнта) — **сірий плейсхолдер-блок із текстовою
  міткою**, напр. `[зона: колір проєкту]`, а не реальне зображення.

## 2. Розмітка — семантичний HTML
- Зони позначаємо **семантичними тегами**, а не суцільними `div`:
  `header` · `nav` · `main` · `section` · `article` · `aside` · `form` · `label` · `input` · `select` · `button` · `ul/li` · `figure/figcaption`.
- Кожна зона = відповідний тег: шапка → `header`, навігація/таби → `nav`, основний вміст → `main`,
  логічний блок → `section`, одиниця списку (проєкт, клієнт, запис) → `article` чи `li`,
  поля вводу → `form` + `label`+`input`, дії → `button`.
- `div` — лише як технічна обгортка для розкладки, не як носій сенсу.

## 3. Текст — справжній доменний, не заглушки
- Реальний текст із домену TrackOn, **не lorem ipsum**:
  - **Проєкти:** «Лендинг для Acme», «Редизайн мобільного застосунку», «Підтримка сайту».
  - **Клієнти:** «Acme Studio», «Іван Петренко», «GLS Logistics».
  - **Записи часу:** «14:00–15:30 · Вёрстка хедера», тривалість «1 год 30 хв», нотатка «правки по фідбеку».
  - **Гроші/ставки:** «$55/год», «UAH», «разом: $320».
  - **Інвойси/статуси:** «не сплачено / відправлено / сплачено».
  - **Стан таймера:** «Активний: Лендинг для Acme · 00:42:10», «Старт».
  - **Фільтри/періоди:** «Тиждень / Місяць / Діапазон», «по проєкту / по клієнту».
- Порожні/помилкові стани теж мають **доменний** текст: «Ще нема проєктів», «Час кінця раніший за початок».

## 4. Назви файлів — латиниця, base + стан
- **Базова сторінка екрана:** `wireframes/<name>.html`
- **Кожен стан — окрема сторінка:** `wireframes/<name>-<state>.html`
- Усе **латиницею**, `<state>` ∈ `empty` · `error` · `loading`. **Успіх = базова** (без суфікса).
- Карта файлів для головного flow (стани — з [`_screens.md`](_screens.md)):

| Екран (sitemap.md) | base (`success`/default) | стани → файли |
|---|---|---|
| Головний / Лаунчпад трекінгу | `tracker.html` | `tracker-empty.html` · `tracker-error.html` · `tracker-loading.html` |
| Форма проєкту (створити / редагувати) | `project-form.html` | `project-form-error.html` · `project-form-loading.html` |
| Клієнт — вибір / створення | `client.html` | `client-empty.html` · `client-error.html` · `client-loading.html` |

> Усього 11 сторінок: 3 базові + 8 станів. `empty` для Форми проєкту **не робимо** —
> `_screens.md` каже, що сценарій його не дає. Окремих `-success` сторінок нема (успіх = базова).

## 5. Стани — окрема сторінка, спільний каркас
- **Кожен стан — окрема HTML-сторінка.** Базова = успіх/дефолт (напр. лаунчпад із проєктами та
  активним таймером); далі `-empty` / `-error` / `-loading`.
- **Та сама семантична структура** на всіх сторінках екрана — змінюється лише **вміст і видимість**
  блоків (порожній список замість заповненого, повідомлення помилки в `form`, скелет/спінер-зона
  замість даних). Каркас не переписуємо — копіюємо і наповнюємо інакше.

## 6. Лишаємо на потім (не у вайрфреймах)
- Колір і акцент бренду, шрифти/типографіка, тіні, заокруглення-як-стиль, **іконки**, ілюстрації.
- Готові UI-компоненти, дизайн-система, мікровзаємодії/анімації, адаптивні брейкпойнти-як-полиш.
- Усе це — наступні кроки; вайрфрейм відповідає лише на «що і де», не «який вигляд».

---

### Каркас-приклад (ілюстрація правил 1–2, НЕ екран)
```html
<header>… ідентифікація екрана, глобальні дії …</header>
<nav>… таби / перемикачі стану …</nav>
<main>
  <section aria-label="…">
    <ul>
      <li><article>… одиниця домену: проєкт / клієнт / запис …</article></li>
    </ul>
  </section>
</main>
```
Сірі блоки, семантичні теги, реальний доменний текст усередині — **без** кольору, шрифтів та іконок.
