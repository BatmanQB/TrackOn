# Research — TrackOn

Дослідження для продукту: користувачі, проблеми, конкуренти, інсайти.

## 1. Проблема

Фрілансери трекають час по кількох клієнтах/проєктах і потребують:
- швидко затрекати час (1–2 тапи),
- бачити витрати часу й грошей,
- звіти за тиждень/місяць/період,
- інвойси й статуси оплати (не сплачено / відправлено / сплачено).

Біль: існуючі трекери або надто складні (для команд), або не покривають інвойси/звіти.

## 2. Цільовий користувач

Фрілансер з кількома клієнтами, працює переважно з телефону, цінує швидкість і простоту.

## 3. Конкуренти

Три групи. До 5 у кожній. Для кожного: тип / чому в групі / що вивчити для нас.

### 3.1 ХАРД — той самий продукт, аудиторія, ринок
Трекери часу для фрілансерів з білінгом/інвойсами.

| Назва | Тип | Чому тут | Що вивчити |
|---|---|---|---|
| **Harvest** | Time tracking + invoicing | Класика «час → ставка → інвойс» для фрілансерів. Прямий аналог ядра | Потік від запису до інвойсу; модель ставок; статуси інвойсу |
| **Clockify** | Free time tracker | Найпопулярніший безкоштовний, велика фрілансерська база, мобільний | Простота старту таймера; мобільне додавання запису; billable vs non-billable |
| **Paymo** | Freelancer PM + time + invoice | Та сама аудиторія, близько до структури Клієнт→Проєкт | Ієрархія клієнт/проєкт/задача; час→інвойс; звіти по проєкту |
| **TimeCamp** | Time tracking + billing | Прямий конкурент: ставки, білабельність, інвойси | Логіка billable-rate та оверрайдів (наш rateOverride); звіти за період |
| **Timely** | Mobile-first / auto tracking | Та сама ніша з акцентом на мобільному — орієнтир для mobile-first | Мобільні патерни швидкого трекінгу; зменшення ручної роботи |

### 3.2 СОФТ — інший продукт, той самий JTBD (трекати/організовувати роботу)
Таск- і проєкт-менеджмент. Не трекають час/гроші, але «тримають роботу під контролем».

| Назва | Тип | Чому тут | Що вивчити |
|---|---|---|---|
| **Trello** | Kanban-дошки | JTBD «бачити статус роботи»; колонки-статуси ≈ наші не сплачено/відправлено/сплачено | Kanban як модель статусів інвойсів; швидке додавання; мобільна простота |
| **Asana** | Task/project mgmt | Організація задач по проєктах — близько до Клієнт→Проєкт→Запис | Структура проєктів без перевантаження; перемикання між проєктами |
| **Jira** | Issue tracking | Еталон «трекінгу» в IT; антиприклад складності для нашої простоти | Де складність вбиває швидкість (що НЕ робити); фільтри/звіти |
| **Notion** | Docs + DB | Гнучкі бази й views; JTBD «все в одному» | Патерни view (таблиця/борд/календар) для звітів |
| **Todoist** | Task manager | Еталон швидкого вводу й мобільної простоти | Quick-add, мінімум тапів — для «затрекати в 1-2 тапи» |

### 3.3 АСПІРАЦІЙНІ — міжнародні еталони категорії трекінгу часу
На кого рівняємось за якістю UX/продукту.

| Назва | Тип | Чому тут | Що вивчити |
|---|---|---|---|
| **Toggl Track** | Time tracking (UX-еталон) | Золотий стандарт frictionless-таймера й чистого UX | One-tap старт/стоп; персистентний таймер; проста подача звітів |
| **Timely (Memory AI)** | Auto/AI time tracking | Еталон «трекінг без зусиль» + преміальний візуал | Автозаповнення дня; візуальна подача часу; light/dark |
| **Timeular** | Tracker + delightful app | Еталон habit-forming UX | Мікровзаємодії, фідбек, мотивація трекати регулярно |
| **RescueTime / Rize** | Automatic insights | Еталон аналітики й інсайтів з часу | Дашборд-метрики, тренди за період — орієнтир для звітів |
| **Sunsama** | Calm productivity | Еталон «спокійного» преміум-дизайну й фокусу | Візуальна мова, темп інтерфейсу, відчуття простоти |

**Логіка розподілу:** Toggl Track свідомо в аспіраційних, а не в ХАРД — для нас це
насамперед UX-еталон категорії, а не вузький конкурент по фрілансерському білінгу.
Hard-група — ті, хто реально б'ється за того ж фрілансера з тим самим «час→інвойс».
