02Ієрархія Jobs

Формат «Коли [ситуація] · я хочу [мотивація] · щоб [результат]». Без назв функцій — лише прогрес людини.

MAIN JOB

Коли я виконую оплачувану роботу для кількох клієнтів, я хочу без зусиль зафіксувати, куди дівся мій час, щоб упевнено отримати за нього справедливу й безсуперечну оплату.

Звідки: усі 3 інтерв'ю + ринок «час→гроші» (research §1.2) + CLAUDE.md §1

04Висновок та чесні слабкі місця

🎯 3 jobs у ядро MVP

Важливі для primary (A) + ринок не закриває.

  1. E2 — не загубитися у складності. Заявлений клин, але з осторогою: Toggl задовольняє 2/3 — гэп вужчий, ніж «❌». Закрити Q2 перед ставкою.
  2. R1 — почати, не втрачаючи фокус. Ринок не дотягує до «1–2 дії».
  3. R4 — час → оплата без зведення. Free-трекери ховають білінг за paywall; щілина = білінг без paywall у простому локальному інструменті.

✂️ Кандидати на виліт

Не закривають жоден job.

  1. Mobile-first / телефон-пріоритет — 3/3 трекають з комп'ютера.
  2. Конвертація валют — ніхто не просив.
  3. Повний фоновий автотрекінг — лише HJ1, архітектурно неможливий у PWA.
  4. Глибокі реквізити / складні поля — годують анти-паттерн E2 («як CRM»).

🔴 Найнебезпечніший список — керують продуктом, стоять на [?] / вигаданому

1. Персона A як PRIMARY. Уся пріоритизація зважена на side-gig дизайнера — бо це перші дані, не бо доведено, що це найцінніший ринок. Стоїть на 2 людях.

2. «Простота — незакритий гэп». Власні дані підточують: I#2 та I#3 задоволені Toggl і звуть його простим. Категорія місцями вже «good enough».

3. «Desktop-first → геть mobile» — вже змінило CLAUDE.md на N=3 з перекосом у дизайнерів.

4. Персона C + «не робити автотрекінг». Тримається на архітектурі (міцно) + вигаданій C + 1 особі (I#2).

5. R4 у ядрі при важливості A=2. Інвойсинг у ядрі, хоча для primary це «дратує», не must-have; тяга гэпу до вибору не доведена.

6. Idle-trim як рішення R2 — у research позначено [?] (Page Visibility API у фоні PWA). Якщо не злетить — функція порожня.
